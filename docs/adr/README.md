# Architecture Decision Records (ADR)

Здесь фиксируются **архитектурные решения** проекта — *почему* выбрали именно такой
вариант, какие альтернативы отвергли и с какими последствиями. ADR дополняют, но не
заменяют другие источники:

| Где | Что хранит |
|---|---|
| **`docs/adr/`** | *Почему* решили так — решение + отвергнутые альтернативы + последствия |
| **`CLAUDE.md`** | *Как теперь писать код* — правило-итог (Domain Rules, Coding Rules) |
| **git history** | *Что* изменилось |

## Когда заводить ADR

Не на каждое изменение. Планка — выполнено хотя бы одно:

- решение **трудно откатить** (схема БД, контракт API, выбор слоя);
- оно **затрагивает несколько модулей** или задаёт паттерн на будущее;
- были **реальные отвергнутые альтернативы**, и без записи их предложат снова.

Баг-фиксы, переименования, косметика и чисто локальные правки — **без ADR**.

## Формат

Лёгкий [MADR](https://adr.github.io/madr/). Один файл = одно решение, нумерация сквозная:
`NNNN-краткий-заголовок.md`. Шаблон — [`0000-template.md`](0000-template.md).

## Жизненный цикл статуса

`Proposed` → `Accepted` → (со временем) `Superseded by ADR-XXXX` / `Deprecated`.
Принятые ADR **не редактируют по существу** — вместо этого пишут новый, заменяющий.
