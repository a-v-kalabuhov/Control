# Автомат состояния ТПА — спецификация

Определяет вычисление `MachineState` (режим, счётчик циклов, исход цикла, поля) из сигналов
USR-шлюза. Реализация: `UsrConnector.Core/MachineStateMachine.cs`. Исполняемая спецификация:
`UsrConnector.Core.Tests/MachineStateMachineTests.cs` — **изменения семантики вносятся только
синхронно в код, тесты и этот документ.**

---

## 1. Контракт: MachineState

| Поле | Тип | Смысл |
|---|---|---|
| `Mode` | enum | `idle` / `auto` / `alarm` / `offline` |
| `CycleCounter` | long | счётчик **износа пресс-формы**: +1 на каждый впрыск, независимо от исхода цикла |
| `LastCycleCompletion` | enum | `NoneYet` / `Normal` / `Aborted` / `Interrupted` |
| `TimestampUtc` | DateTimeOffset | момент опроса |
| `Fields` | словарь | опциональные поля: `cushion`, `lastCycleDurationMs`, `reject`, `moldPosition`, `injectionPosition`, поля шаблона (чиллер и т.п.) |

Принцип: **коннектор снимает и отдаёт, интерпретируют внешние системы.** Поэтому подушка
публикуется при любом исходе цикла; фильтрация по `LastCycleCompletion` — на стороне
потребителя. `manual` в enum отсутствует (решение: без впрыска различить наладку и простой
на минимальном наборе сигналов нельзя — всё без впрыска есть `idle`).

## 2. Ключевые решения

**Якорь цикла — впрыск (`Injection ↑`).** `Mould closed` неоднозначен (форма смыкается и в
ручных операциях); впрыск однозначно означает производственный цикл. На фронте впрыска:
`CycleCounter++` (форма нагружена — износ состоялся, что бы ни случилось дальше),
`Mode = auto`, открывается окно цикла.

**Детекция впрыска — дискретный сигнал** (Injection 1, DO14 KEBA через оптопару), не порог
по аналогу: порог смещения шнека требует технологической калибровки под каждую машину
(доза/материал/изделие) — overengineering для центрального сигнала модели.

**Штатное завершение — `EjectorFwdReached ↑` (E1).** Защёлкиваются: подушка, длительность,
reject. `Mode` остаётся `auto` до idle-таймаута.

**Подушка = минимум `InjectionPosition` за окно цикла** (от `Injection ↑` до завершения).
Мгновенный замер неверен: точный момент спада Injection между опросами не виден
(дискретизация), а к E1 шнек уже набрал новую дозу. Минимум — экстремум траектории,
устойчивый к сетке опроса. Публикуется при любом исходе. Для двухузловой машины (2K)
аналогично трекается `InjectionPosition2` за то же окно (узлы работают синхронно) —
поле `cushion2`.

**Idle и Alarm взаимоисключающие по построению:**
- впрыск был (окно цикла открыто) → возможен только **alarm** (E1 не наступил за таймаут);
- впрыска не было → возможен только **idle** (впрыск не пришёл за таймаут от конца
  последнего цикла).

**Таймауты адаптивные:** скользящее среднее длительности последних N **успешных auto-циклов**
× коэффициент (`alarmCoef`, `idleCoef` — в конфиге). До статистики — `seedCycleMs`
(консервативные 300 с: реальные циклы 3…300 с, единый рациональный дефолт невозможен —
первый зависший цикл быстрой машины детектируется с запозданием, зато без ложных тревог).
Средне учитывает **только** циклы, завершённые на E1. Сброс статистики при простое дольше
15 мин (вероятная переналадка — сменилась форма, сменилась длительность).

**Offline:** N подряд неудачных опросов (`OfflineAfterFailedPolls`; демпфер против ложного
Interrupted на единичной сетевой ошибке — суммарное время порога рекомендуется сравнимым с
длительностью цикла). Активный цикл помечается `Interrupted`. Восстановление связи → `idle`;
детекторы фронтов пересеваются с первого снимка (ложные фронты «через обрыв» не ловятся).
Верхний слой в offline не отправляет сообщения наружу.

## 3. Таблица переходов

| Событие | Условие | Действия |
|---|---|---|
| Старт коннектора | — | `mode=idle`, `counter=0`, `completion=NoneYet` |
| `Injection ↑` | — | `counter++`; `mode=auto`; открыть окно (сброс `cushionMin`); если цикл был активен — прежний закрыт `Aborted` |
| каждый опрос | окно открыто | `cushionMin = min(cushionMin, InjectionPosition)` |
| `EjectorFwdReached ↑` | окно открыто | закрыть `Normal`; защёлкнуть cushion/duration/reject; обновить среднее; `mode=auto` |
| таймаут alarm | окно открыто, `t_от_впрыска > avg×alarmCoef` | закрыть `Aborted` (cushion защёлкивается, duration — нет); `mode=alarm` |
| таймаут idle | окна нет, `t_от_конца_цикла > avg×idleCoef` | `mode=idle` |
| неудачный опрос ×N | — | активный цикл → `Interrupted`; `mode=offline` |
| восстановление связи | был offline | `mode=idle`; пересев фронтов |
| простой > 15 мин | окна нет | сброс статистики среднего (→ seed) |

Выход из `alarm`: следующий штатный цикл (E1 → auto), idle-таймаут (→ idle) или обрыв связи
(→ offline).

## 4. Минимально необходимые роли сигналов

| Роль | Обязательность | Назначение |
|---|---|---|
| `Injection` | **обязательна** | якорь цикла, auto, счётчик износа |
| `EjectorFwdReached` | **обязательна** | штатное завершение (E1) |
| `InjectionPosition` | нужна для подушки | минимум за окно = cushion |
| `Reject` | опциональна | вердикт качества в поле `reject` |
| `MouldClosed`/`MouldOpened`/`EjectorBwdReached` | опциональны | публикация/диагностика; в автомате не участвуют |
| `MoldPosition` | опциональна | публикация текущего положения плиты |

Без `Injection` и `EjectorFwdReached` автомат неработоспособен — валидация профиля должна
требовать их назначения.
