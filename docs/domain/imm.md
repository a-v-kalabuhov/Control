# ТПА — термопластавтомат (`Imm`)

> Литьевая машина: смыкает пресс-форму, впрыскивает расплав, формирует деталь.
> Главный объект мониторинга всей системы.

## Что это

Термопластавтомат (ТПА, англ. injection molding machine — отсюда `Imm`) — единица
производственного оборудования. На ТПА устанавливают пресс-форму ([`mold.md`](mold.md)) и
выполняют сменное задание ([`shift-task.md`](shift-task.md)). Машина шлёт телеметрию по MQTT
(циклы, режим работы), на основе которой система ведёт мониторинг, считает выпуск и простои.

С ТПА работают: наладчик (ставит ПФ, налаживает, ведёт задание), менеджер (выдаёт задания,
смотрит дашборд/отчёты).

## Код

- Сущность: `Wintime.Control.Core.Entities.Imm`
- Связанные: `Template` (конфиг датчиков), `ShiftTask`, `ImmCycle`, `Event`, `ImmStatusHistory`

## Ключевые поля

| Поле | Смысл и правила |
|---|---|
| `Name` | Заводское обозначение машины (видит пользователь) |
| `TemplateId` | Шаблон оборудования — какой набор датчиков/режимов ожидать от этой машины |
| `ConnectorAlias` | Псевдоним машины в дереве OPC-сервера (для коннекторов; см. скил `connector-integration`) |
| `IsActive` | **Архивный флаг**, не «работает сейчас». `false` → выведена из оборота (см. инварианты) |

## Состояния (сырой статус и эффективное состояние)

У ТПА два разных понятия «состояния» — подробно в [`imm-status.md`](imm-status.md):

- **Сырой статус** (`ImmStatusHistory.Status`, по MQTT): `Auto` · `Manual` · `Idle` ·
  `Alarm` · `Offline`. Единственная точка записи — `IImmStatusService.UpdateStatusAsync`.
- **Эффективное состояние** (для дашборда, **вычисляется**, не хранится): сводит сырой режим
  + статус активного задания + открытый простой. См. ADR-0006.

## Инварианты (нарушение = баг)

- **`IsActive` — мягкое удаление, не текущая активность.** `false` → скрыт из списков выбора,
  нельзя назначить в новое СЗ; история сохраняется и доступна в отчётах. Физически ТПА
  не удаляют никогда. *Почему:* нужно сохранять историческую выработку/простои выбывшей машины.
  → [CLAUDE.md], раздел «`IsActive`».
- **Статус пишется только через `UpdateStatusAsync`** (обновляет и БД, и кеш `IImmStatusCache`).
  Мимо этой точки статус менять нельзя — иначе кеш и история разойдутся.
